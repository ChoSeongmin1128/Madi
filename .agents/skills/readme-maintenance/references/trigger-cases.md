# Trigger Cases

## Trigger 되어야 하는 요청

- 이번 변경이면 README도 고쳐야 하나요
- README에 현재 제품 방향을 어떻게 적어야 하나요
- 사람용 문서에 어떤 내용만 남기는 게 좋을까요
- README가 지금 사실과 맞는지 점검해 주세요

## Trigger 되지 않아야 하는 요청

- 커밋 메시지 형식을 정해 주세요
- 저장 포맷을 어떻게 설계할까요
- pnpm 말고 다른 패키지 매니저로 바꿔도 되나요

## 기대 동작

- README 갱신 여부를 “의미 있는 변경” 기준으로 판단합니다.
- README에 넣지 말아야 할 AI 전용 규칙을 걸러냅니다.
- README나 관련 문서 구조 개선은 사용자 승인 없이 확정하지 않습니다.
