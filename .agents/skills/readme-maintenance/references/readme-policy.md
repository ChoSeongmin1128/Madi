# README Policy

## 목적

README는 사람에게 현재 저장소의 목적, 상태, 진입점을 빠르게 설명하는 문서입니다. AI 전용 세부 규칙은 skill 문서에 두고, README는 사람이 읽을 최소한의 안내에 집중합니다.

## 언제 README를 갱신하는가

다음처럼 의미 있는 변경이 있을 때 README 갱신을 검토합니다.

- 설치, 실행, 빌드 방식이 바뀔 때
- 제품 방향이나 핵심 개념이 바뀔 때
- 저장소 진입 방법이나 문서 구조가 바뀔 때
- 사용자가 알아야 할 스크립트나 워크플로가 추가될 때
- 배포 규칙, 태그 규칙, 릴리스 흐름이 바뀔 때
- 아직 구현은 없더라도 현재 프로젝트 상태를 설명하는 문장이 틀리게 될 때

## 언제 README를 갱신하지 않는가

- 내부 리팩터링만 있을 때
- 사람용 README보다 skill 문서가 source of truth인 AI 전용 규칙만 바뀔 때
- 문장만 조금 더 예쁘게 고치려는 정도일 때

## 작성 원칙

- README는 짧고 사람이 읽기 쉽게 유지합니다.
- 상세 규칙은 README에 반복하지 말고 관련 skill로 보냅니다.
- 현재 상태와 제품 방향이 틀리지 않도록 유지합니다.
- 아직 확정되지 않은 기술 선택을 README에 사실처럼 적지 않습니다.

## Gotchas

- README를 “모든 규칙의 저장소”로 만들지 않습니다.
- AI 전용 운영 규칙을 README에 길게 복제하지 않습니다.
- README, AGENTS, skill 문서를 더 잘 정리할 아이디어가 생겨도 먼저 사용자에게 질문합니다.
