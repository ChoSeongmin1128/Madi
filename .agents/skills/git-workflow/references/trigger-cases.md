# Trigger Cases

## Trigger 되어야 하는 요청

- 이 변경에 맞는 커밋 메시지 써주세요
- 브랜치 이름을 어떻게 잡는 게 좋을까요
- PR 제목과 본문을 정리해 주세요
- 변경 사항을 리뷰어가 보기 좋게 요약해 주세요

## Trigger 되지 않아야 하는 요청

- 이 앱의 저장 포맷을 어떻게 설계할까요
- macOS 앱 스택을 무엇으로 정할까요
- 이번 변경이면 README도 갱신해야 하나요

## 기대 동작

- 커밋/브랜치/PR 형식과 예시를 우선 제공합니다.
- 저장소 Git 규칙을 바꾸는 결정은 사용자 승인 없이 확정하지 않습니다.
