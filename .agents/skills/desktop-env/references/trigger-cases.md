# Trigger Cases

## Trigger 되어야 하는 요청

- 이 레포는 npm이 아니라 pnpm을 써야 하나요
- macOS 전용 전제를 어디까지 두고 설명해야 하나요
- 설치나 실행 명령 예시는 어떤 기준으로 적어야 하나요
- 아직 스택이 안 정해진 상태에서 환경 문서를 어떻게 써야 하나요

## Trigger 되지 않아야 하는 요청

- 블록 기반 제품 구조를 어떻게 잡을까요
- Markdown 편집 UX 원칙을 어떻게 둘까요
- 커밋 메시지 형식을 정해 주세요

## 기대 동작

- 확정된 환경 사실만 설명합니다.
- 미정 스택을 확정처럼 쓰지 않습니다.
- 환경 규칙 변경이 필요해 보이면 먼저 사용자에게 질문합니다.
