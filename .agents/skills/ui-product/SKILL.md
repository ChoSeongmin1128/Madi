# UI Product

이 skill은 MinNote의 편집 경험과 제품 상호작용 원칙을 다룹니다.

다음 상황에서 먼저 사용합니다.
- 블록 UI나 편집 인터랙션을 설계할 때
- Markdown 입력 경험을 정의할 때
- Heynote-like, Notion-like 균형을 판단할 때
- 새 UI 요소가 단순함을 해치는지 검토할 때

다음 상황에서는 이 skill을 우선 사용하지 않습니다.
- 커밋/브랜치/PR 규칙만 정리할 때
- 설치/실행 환경이나 패키지 매니저 정책만 확인할 때
- 저장 포맷, 데이터 모델, 런타임 선택 같은 큰 구조 결정을 내릴 때

세부 원칙과 금지 패턴은 `references/ui-principles.md`를 기준으로 따릅니다.
트리거 예시는 `references/trigger-cases.md`를 기준으로 따릅니다.

주의:
- 일반적인 “풍부한 에디터 기능”보다 제품의 단순함과 블록 구분감을 우선합니다.
- UI 원칙을 보강하거나 바꿔야 할 것 같으면 먼저 사용자에게 질문합니다.
