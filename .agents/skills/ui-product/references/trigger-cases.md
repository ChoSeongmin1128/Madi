# Trigger Cases

## Trigger 되어야 하는 요청

- 블록 경계를 어떻게 보여줘야 할까요
- Markdown 체크리스트 입력 경험을 어떻게 잡을까요
- Notion식 편집과 Heynote식 블록 구분을 어떻게 섞을까요
- 이 UI가 너무 무거워 보이는지 검토해 주세요

## Trigger 되지 않아야 하는 요청

- 브랜치 이름 정해 주세요
- pnpm 기준 실행 명령을 알려 주세요
- 저장 포맷 스키마를 설계해 주세요

## 기대 동작

- 편집 경험과 상호작용 원칙을 우선 다룹니다.
- 큰 구조 결정은 `product-architecture`와 경계를 구분합니다.
- 새로운 UI 원칙 추가나 수정은 사용자 승인 대상으로 봅니다.
