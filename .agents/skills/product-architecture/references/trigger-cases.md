# Trigger Cases

## Trigger 되어야 하는 요청

- 저장 포맷을 어떻게 잡는 게 좋을까요
- 상위 블록 구조와 Markdown 편집기를 어떻게 나눌까요
- 기존 앱을 수정할지 새로 만들지 판단해 주세요
- 블록 모델과 데이터 모델을 어떻게 설계할까요

## Trigger 되지 않아야 하는 요청

- 커밋 메시지 써 주세요
- README를 이번에 꼭 갱신해야 하나요
- 버튼 문구를 더 자연스럽게 바꿔 주세요

## 기대 동작

- 확정된 전제와 미정 항목을 구분해서 설명합니다.
- 구조적 의사결정을 임의로 확정하지 않습니다.
- 되돌리기 비싼 선택은 사용자 승인 대상으로 봅니다.
